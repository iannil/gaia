## 1. 工具箱管理

### 克制原则

> 仅在绝对必要时，才将依赖项打包成 Skill。只添加本任务周期内一定会用到的工具。

### 判断标准

| 标准 | 说明 |
|------|------|
| 绝对必要 | 不用它无法完成任务 |
| 本周期使用 | 本次任务就会用到，不是"未来可能" |
| CLI/API 友好 | 命令行或 API 调用，而非 GUI 重型 |
| 可验证效果 | 能明确验证它解决了问题 |

### 适用性判断

使用 `github-to-skills` 前，评估项目是否适合：

| 特征 | 适合 | 不适合 |
|------|------|--------|
| 交互方式 | CLI/API | GUI/复杂环境 |
| 依赖复杂度 | 低 | 高（需要特定环境） |
| 输出形式 | 文本/JSON | 富媒体/可视化 |
| 安装难度 | 简单 | 复杂配置 |

**决策**: 如果不适合，考虑替代方案（如手动操作、其他工具）。

### 实践要点

- ❌ "这个库很厉害，先打包成 Skill 备用"
- ❌ "未来可能用到，先加上"
- ✅ "这个任务必须用它，现在打包"
- ✅ "只用它的核心功能，不需要全部打包"

## 2. 指令映射

### 目的
建立个人化的指令快捷表，简化常用操作的调用。

### 基础映射表

| 简写指令 | 完整指令 | 说明 |
|----------|----------|------|
| `ls-skills` | `列出所有 Skills` | 查看已安装的 Skills |
| `check-skills` | `检查所有 Skills 的更新` | 更新检查 |
| `search-skill` | `在可信源中搜索 Skill` | Phase G 路径一 |
| `skill-from-github` | `从 GitHub 学习并构建 Skill` | Phase G 路径二 |
| `skill-from-masters` | `从专家方法论构建 Skill` | Phase G 路径三 |

### 自定义映射

根据个人使用习惯添加：

```markdown
## 个人指令映射

| 简写 | 完整指令 | 使用场景 |
|------|----------|----------|
| `evolve` | `将本次使用 [Skill] 的经验记录到 evolution.json` | 任务完成后 |
| `clean-skills` | `清理临时 Skills` | 验收阶段 |
```

### 管理方式

1. 维护在 CLAUDE.md 或个人笔记中
2. 根据使用频率更新
3. 定期清理不再使用的映射

## 3. 记录即演化

### 核心思想

> 任何有效的参数、成功的技巧、遇到的坑点，都必须记录到对应 Skill 的 `evolution.json` 中。

### evolution.json 结构

```json
{
  "skill_name": "skill-name",
  "version": "1.0.0",
  "last_updated": "2025-01-31",
  "usage_history": [
    {
      "date": "2025-01-30",
      "context": "用于批量图片转 ASCII",
      "effective_params": {
        "resolution": "100x100",
        "charset": "standard"
      },
      "lessons_learned": [
        "分辨率过高会导致输出混乱",
        "标准字符集在大多数场景下足够"
      ]
    }
  ],
  "pitfalls": [
    {
      "issue": "处理透明背景图片时出错",
      "solution": "先添加白色背景层"
    }
  ],
  "best_practices": [
    "先处理小图验证效果",
    "使用 --preview 参数快速预览"
  ]
}
```

### 记录指令模板

```
"将本次使用 [Skill名] 的经验 '[具体经验]' 记录到进化日志中。"
```

### 应记录的内容

| 类型 | 示例 |
|------|------|
| 有效参数 | `--threads 4` 效果最好 |
| 成功技巧 | 先用小数据集测试 |
| 坑点 | 路径包含空格会失败，需加引号 |
| 依赖条件 | 需要 Python 3.10+ |
| 性能观察 | 处理 1MB 文件约需 2 秒 |

### 实践要点

- **立即记录，不要依赖记忆**
- 记录"为什么"，不只是"是什么"
- 定期回顾演化记录，提炼通用规则
- 在 Phase A 验收时更新记录

### 反模式警示

- ❌ "这个参数下次记住就行"
- ❌ "任务完成了，不用管细节"
- ✅ "将坑点记录到 evolution.json"
- ✅ "定期回顾，提炼最佳实践"
