# GAIA 框架总览

## 协议使命 (Mission Statement)

本协议旨在将我们的协作从简单的问答模式，升级为一套系统性的、可演化的复杂问题解决框架。你，将不仅仅是执行者，更是遵循此协议的分析师、架构师和工匠。我们的目标是高效、稳健地生成、分析、实现并验收复杂系统。

## 框架定位

GAIA（Generate → Analyze → Implement → Acceptance）是一个面向 AI 协作的问题解决框架，旨在为复杂任务提供系统化的执行路径。

### 核心价值

- **系统性**：将模糊需求转化为清晰、可执行的方案
- **可演化**：每次交互都沉淀为经验，优化未来决策
- **克制原则**：不为工具而工具，所有行动源于真实需求

### 架构说明

- **核心层**: GAIA 执行引擎，实现 G→A→I→A 状态机
- **Skill 层**: 可复用技能的发现、管理和演化
- **知识层**: 经验沉淀、模板库和最佳实践
- **集成层**: MCP 协议支持和外部系统集成
- **交互层**: CLI、Web 界面和 IDE 插件

详见 [功能模块规划](../02-progress/roadmap.md)

## 四阶段概览

| 阶段 | 思维模式 | 核心目标 | 关键产出 |
|------|----------|----------|----------|
| **G: 生成** | 专家思维 | 定义"做什么"与"怎么做" | 解决方案雏形 |
| **A: 分析** | 架构师思维 | 分解与规划 | MVP 定义与优先级 |
| **I: 实现** | 工匠思维 | 构建与记录 | 可交付成果 |
| **A: 验收** | 质检员思维 | 验证与复盘 | 验收报告与演化记录 |

## 适用场景

GAIA 框架适用于：

- 复杂技术问题的系统性解决
- 需要多步骤决策的开放性问题
- 需要沉淀经验和最佳实践的任务
- 涉及多种工具和依赖的项目
