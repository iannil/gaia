# 术语表

本文档解释 GAIA 框架中的关键术语。

---

## GAIA 核心术语

| 术语 | 英文 | 定义 |
|------|------|------|
| GAIA 框架 | GAIA Framework | Generate → Analyze → Implement → Acceptance，一个面向 AI 协作的问题解决框架 |
| Phase G | Phase Generate | GAIA 第一阶段：生成，定义"做什么"与"怎么做" |
| Phase A | Phase Analyze | GAIA 第二阶段：分析，分解与规划 |
| Phase I | Phase Implement | GAIA 第三阶段：实现，构建与记录 |
| Phase A | Phase Acceptance | GAIA 第四阶段：验收，验证与复盘 |
| MVP | Minimum Viable Product | 最小可用产品，定义核心功能边界 |
| P0-P3 | Priority Levels | 优先级等级：P0(阻塞) > P1(核心) > P2(重要) > P3(锦上添花) |

---

## Skill 相关术语

| 术语 | 定义 |
|------|------|
| Skill | 可复用的技能包，包含工具、配置和使用经验 |
| evolution.json | Skill 的演化记录文件，记录使用历史、坑点、最佳实践 |
| github-to-skill | 从 GitHub 仓库创建 Skill 的工具 |
| skill-manager | Skill 管理工具，用于安装、更新、清理 Skills |
| ls-skills | 列出所有已安装 Skills 的简写指令 |
| check-skills | 检查 Skills 更新的简写指令 |

---

## 核心原则术语

| 术语 | 定义 |
|------|------|
| 需求驱动 (Need-Driven) | 所有行动必须源于明确、真实的需求 |
| 站在巨人肩上 (Leverage Giants) | 优先利用现有市场/社区方案 |
| 工匠与专家结合 (Craftsman & Expert Duality) | 宏观专家思考，微观工匠工作 |
| 动态演化 (Dynamic Evolution) | 每次交互记录并沉淀，优化未来决策 |

---

## Phase G 路径术语

| 术语 | 定义 |
|------|------|
| 市场优先 (search-skill) | Phase G 路径一：优先搜索现成 Skills |
| 原理优先 (skill-from-github) | Phase G 路径二：从 GitHub 学习核心算法 |
| 方法论优先 (skill-from-masters) | Phase G 路径三：综合专家理论构建方案 |
| Tier 1/2/3 | 技能来源的信任层级 |

## 其他术语

| 术语 | 定义 |
|------|------|
| 克制原则 | 只安装和使用绝对必要的工具 |
| 文件即记忆 | 文档和代码是协作记忆的载体 |
| 排名优于分数 | 优先级排序比时间估算更重要 |
| 分数守底线 | 验收时对照核心需求进行 Pass/Fail 检查 |
| 复盘 | 任务完成后的回顾与经验沉淀 |
| 反模式 | 应该避免的常见错误做法 |
