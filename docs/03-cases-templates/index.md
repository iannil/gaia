## 案例

### 已收录案例

当前暂无实际案例。案例将在第一个实际项目完成后添加。

### 案例提交

如果你使用 GAIA 框架完成了项目，欢迎提交案例。一个完整的案例应包含：

1. **项目背景**: 需求描述、约束条件
2. **Phase G**: 采用了哪条路径，如何找到方案
3. **Phase A**: MVP 定义、优先级排序、风险评估
4. **Phase I**: 实现过程、使用的 Skills、遇到的坑点
5. **Phase A (验收)**: 验收结果、复盘结论

## 案例模板

```markdown
## [项目名称]

### 背景
- **需求**: [描述]
- **约束**: [描述]
- **时间**: [日期范围]

### Phase G: 生成
- **采用路径**: [市场优先/原理优先/方法论优先]
- **搜索结果**: [描述]
- **最终方案**: [描述]

### Phase A: 分析
- **MVP**: [列出核心功能]
- **优先级**: [P0/P1/P2 划分]
- **风险**: [主要风险和缓解]

### Phase I: 实现
- **使用的 Skills**: [列表]
- **实现过程**: [简要描述]
- **坑点与解决**: [记录]

### Phase A: 验收
- **验收结果**: [Pass/Fail]
- **复盘结论**: [成功和改进]

### 交付物
- [链接到代码/文档]

### 经验总结
- [对未来项目的建议]
```
