# Phase 启动模板

本模板用于 GAIA 流程中各阶段的启动检查和初始化。

---

## 使用说明

在每个 Phase 开始前，使用此模板进行快速检查和初始化。

---

## Phase 特定检查

### Phase G: 生成 (Generate)

**核心问题**: 我们要解决什么问题？

**检查项**:
- [ ] 问题陈述清晰
- [ ] 已尝试搜索现有解决方案
- [ ] 已确定解决方案路径（市场/原理/方法论）

**三条路径选择**:
- [ ] `search-skill`: 从市场寻找现成 Skill
- [ ] `skill-from-github`: 从 GitHub 仓库学习
- [ ] `skill-from-masters`: 综合专家理论

---

### Phase A: 分析 (Analyze)

**核心问题**: 如何分解和规划？

**检查项**:
- [ ] MVP 定义清晰（核心功能 vs. 锦上添花 vs. 明确不做）
- [ ] 子任务已按优先级排序
- [ ] 技术依赖已识别

**输出结构**:
```
P0 (必须):
- [ ] 任务 1
- [ ] 任务 2

P1 (重要):
- [ ] 任务 3

P2 (可选):
- [ ] 任务 4
```

---

### Phase I: 实现 (Implement)

**核心问题**: 如何构建和记录？

**检查项**:
- [ ] 理解现有代码后再修改
- [ ] 遵循"克制原则"：只做必要的事
- [ ] 将有效参数和经验记录到 evolution.json

**工具箱**:
- [ ] `skill-manager`: 打包必要工具
- [ ] `github-to-skills`: 从 GitHub 创建 Skill
- [ ] `skill-evolution-manager`: 记录演化经验

---

### Phase A: 验收 (Acceptance)

**核心问题**: 如何验证和复盘？

**检查项**:
- [ ] 对照 Phase A 的 MVP 定义进行 Pass/Fail 验证
- [ ] 复盘：更新 evolution.json
- [ ] 清理临时 Skills
